# Backward - from

## 1. Introdução
<p style="text-align: justify;">Utilizamos a técnica de pós-rastreabilidade chamada Backward-from para conseguir mapear os requisitos com suas respectivas fontes. Em outras palavras, serve para listar cada requisitos mapeado com a técnica de elicitação da qual ele surgiu.
</p>

## 2. Metodologia

<p style="text-align: justify;">Pela necessidade de se rastrear a origem dos requisitos levantados ao longo do semestre para o nosso projeto, estamos disponibilizando uma maneira simples e clara de se rastrear as origens de cada requisitos.
</p>
<p style="text-align: justify;">Na tabela de requisitos elicitados, para cada requisito temos uma coluna que especifica quais documentos e técnicas de origens, bem como um link direto de navegação para cada documento. Também está presente o stakeholder que é nada mais do que as partes interessadas. Para melhor rastreabilidade, as técnicas em que cada requisito se originou serão identificadas com a sigla da técnica juntamente com o número do requisito no devido documento. As técnicas serão referenciadas pelas seguintes siglas:
</p>
<center>

| Técnica | Sigla |
|:-:|:-:|
| Brainstorm   | BT |
| Observação   | OBS |
| Storyboard | SB |
| Observação Participativa  | OB  |
| Introspecção | ITP |

</center>
